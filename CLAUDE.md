## Project Overview

**wxc_sdk** is a Python SDK for Webex APIs, with special focus on Webex Calling endpoints. It provides both synchronous (`WebexSimpleApi`) and asynchronous (`AsWebexSimpleApi`) client classes built on `requests` / `aiohttp` and Pydantic 2.x models.

## Package Manager

This project uses **uv**. Prefer `uv run <command>` over plain invocations.

## Common Commands

```bash
# Install dependencies
uv sync

# Run tests
uv run pytest
uv run pytest tests/test_pydantic.py          # single file
uv run pytest tests/test_pydantic.py::TestPydantic::test_something  # single test

# Lint & format
uv run ruff format .
uv run ruff check .
uv run mypy .

# Code generation (make targets wrapping scripts in script/)
make types      # regenerate wxc_sdk/all_types.py
make async      # regenerate wxc_sdk/as_api.py
make docs       # build Sphinx HTML docs
make oas        # regenerate Python from OpenAPI specs
```

# Auto generating Python code from OpenAPI Specs

script/oas2py.py is used to transform OpenAPI specs into Python Code. scripts/oas2py.env defines the base paths for 
the OAS specs and the target directory for the generated Python code.

## Locating OAS spec files

The script resolves input spec files as follows:

1. The `--oas` argument (or the `OAS_SPEC_PATH` environment variable from `oas2py.env`) provides a base path.
2. The basename of `--oas` controls the glob pattern used to find `spec.json` files:
   - Omitted or `**`: matches all specs recursively (`**/spec.json`).
   - A plain API name with no wildcards (e.g. `people`): expands to `**/people/**/spec.json`, matching the
     standard OAS directory layout `<hierarchy>/<api-name>/v1/spec.json`.
   - If a directory part is also given, it is resolved first against the filesystem and then against
     `OAS_SPEC_PATH` as a relative sub-path.
3. The resolved glob is expanded with `glob.glob(..., recursive=True)` and the resulting list is sorted
   alphabetically. Files whose basenames match the `--exclude` regex are removed.

The expected on-disk structure for a spec is:
```
<OAS_SPEC_PATH>/<hierarchy>/<api-name>/v1/spec.json
```
For example:
```
.../openapi/production/Shared/people/v1/spec.json
```

## Determining the output path

Each spec produces one `*_auto.py` file. The output location is chosen as follows:

1. **`--pysrc -`**: output is written to stdout.
2. **`--pysrc <file>`**: written to exactly that path.
3. **Auto-derived name** (no `--pysrc`):
   - The Python basename is taken from the spec path components:
     - Version `v1` → `<api-name>_auto.py` (e.g. `people_auto.py`)
     - Other version (e.g. `v2`) → `<api-name>_v2_auto.py`
   - The output directory is determined by `--pypath` (or `OAS_PY_PATH` from the env file).
   - **Hierarchy mirroring** (enabled when `--pypath` ends with `/`): the directory structure between
     `OAS_SPEC_PATH` and the spec file is recreated under the output path, omitting the last three
     components (`<api-name>/v1/spec.json`). Missing intermediate directories are created automatically.
     Example: spec at `.../production/Shared/people/v1/spec.json` → output at `<OAS_PY_PATH>/Shared/people_auto.py`.
   - **Flat output** (no trailing `/`): all generated files land directly in the output directory.


## Architecture

### Main API class

`wxc_sdk/__init__.py` — `WebexSimpleApi` is a `@dataclass` that aggregates ~30 sub-API objects (one per Webex feature area). Instantiate it with a bearer token:

```python
api = WebexSimpleApi(tokens="Bearer …")
```

The async counterpart lives in the auto-generated `wxc_sdk/as_api.py` as `AsWebexSimpleApi`.

### Layer structure

| Layer | Key files | Purpose |
|---|---|---|
| HTTP | `rest.py`, `as_rest.py` | Session, retries, error handling, pagination |
| Models | `base.py` | `ApiModel` (Pydantic BaseModel), `SafeEnum` |
| Feature APIs | `wxc_sdk/<feature>/` | One package/module per Webex API area |
| Auto-generated | `all_types.py`, `as_api.py` | Never edit manually; regenerate with `make` |

### Telephony

The largest feature area lives in `wxc_sdk/telephony/` (46 sub-modules). It is exposed as `api.telephony` and contains everything Webex Calling related (call queues, hunt groups, auto-attendants, dial plans, voicemail, etc.).

### Pydantic models

All API request/response objects are Pydantic v2 models that inherit from `ApiModel` in `base.py`. `SafeEnum` (from `aenum`) is used for enumerations so unknown values from the API don't raise validation errors.

### Auto-generated code

- `wxc_sdk/all_types.py` — re-exports every model; generated by `script/all_types.py`
- `wxc_sdk/as_api.py` — async wrapper; generated by `script/async_gen.py`
- `open_api/generated/` — Python stubs generated from OpenAPI specs via `script/oas2py.py`

These files are excluded from ruff and mypy checks. **Do not edit them directly.**

### Code style

- Line length: 120
- Single quotes (ruff enforces)
- Strict mypy (exceptions configured in `pyproject.toml` for auto-generated files and tests)
- Conventional commits (commitizen)

## Testing

Tests live in `tests/` and rely on live Webex credentials from a `.env` file. The `tests/base.py` module provides `TestCaseWithLog` and `TestCaseWithUsers` base classes used throughout. Many tests are integration tests that hit real Webex APIs, so a configured `.env` with valid tokens is required for most test runs.

## Adding a New API Endpoint

1. Create or extend the relevant module under `wxc_sdk/<feature>/`.
2. Define Pydantic models inheriting from `ApiModel`.
3. Add the API class attribute to `WebexSimpleApi` in `wxc_sdk/__init__.py`.
4. Run `make types` and `make async` to regenerate the auto-generated files.
